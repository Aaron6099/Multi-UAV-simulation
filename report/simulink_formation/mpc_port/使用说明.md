# Simulink MPC 编队验证 · 傻瓜使用说明

> 不需要了解 ROS2、PX4、Ubuntu。只需要 MATLAB + Simulink + Aerospace Blockset。

---

## 这个代码是干什么的？

用 MATLAB/Simulink 模拟多架无人机一起飞行，验证控制算法是否稳定。

每架无人机都有一个"大脑"（MPC 控制器），负责计算下一步往哪飞。
模拟结束后自动给出**成绩单**：飞得好（PASS）、一般（REVIEW）、还是差（FAIL）。

## 第二步：选你想跑的场景，直接运行对应文件

### 按飞机数量选：

| 文件名前缀 | 飞机数量 | 队形 |
|---|---|---|
| `run_mpc_solo1_` | **1 架** | 单机 |
| `run_mpc_pair2_` | **2 架** | 前后纵列，间距 3m |
| `run_mpc_trio3_` | **3 架** | 等边三角形，间距约 5.2m |
| `run_mpc_cross5_` | **5 架** | 十字形，间距 3m |

### 按飞行方式选：

| 文件名后缀 | 飞行方式 | 说明 |
|---|---|---|
| `_hover` | **悬停** | 飞到指定高度，原地不动 |
| `_line` | **直线** | 向北飞 20 米后停下 |
| `_circle` | **圆周** | 绕半径 10 米的圆飞行 |

### 组合示例：

```matlab
% 想看 2架无人机 飞圆周？
run_mpc_pair2_circle

% 想看 5架无人机 飞直线？
run_mpc_cross5_line

% 想看 单架无人机 悬停？
run_mpc_solo1_hover

% 想看 3架无人机 飞圆周？
run_mpc_trio3_circle
```

直接在 MATLAB 命令窗口输入文件名回车即可，**不需要加任何参数**。

## 第四步：看结果

### 成绩单（文字）

| 成绩 | 含义 | 是否正常 |
|---|---|---|
| **PASS** | 飞得好，误差小 | ✅ 优秀 |
| **REVIEW** | 有点落后，但队形整齐 | ✅ 正常（直线/圆周固有滞后） |
| **FAIL** | 队形散了或出了安全问题 | ❌ 有问题 |

> **REVIEW 不是错误！** 直线和圆周飞行时，所有无人机会整体落后参考点约 1 米，
> 但彼此之间保持队形，这是算法特性，不是故障。

### 成绩表（汇总）

所有跑过的场景成绩都记录在：
```
mpc_port/results.csv
```
用 Excel 打开即可查看。

### 轨迹图（PNG）

每次仿真自动生成一张四格图：
```
report/figures/mpc_<队形>_<飞行方式>.png
```
例如：`mpc_pair2_circle.png`

图中四个格子分别显示：
- 左上：俯视飞行轨迹（圆周模式有灰色参考圆）
- 右上：编队误差随时间变化
- 左下：高度随时间变化
- 右下：最小间距随时间变化（多机时）/ 跟踪误差（单机时）

## 数字指标是什么意思？

```
form_err  = 编队误差（米）  ← 无人机之间的间距是否正确？目标 < 0.3m
track_err = 跟踪误差（米）  ← 飞行路径是否跟上了参考轨迹？目标 < 0.5m(PASS)
alt_err   = 高度误差（米）  ← 高度是否稳定在 -5m？目标 < 0.3m
min_sp    = 最小间距（米）  ← 无人机之间最近距离？须 > 1.5m（安全线）
```

## 如果想修改参数

所有参数集中在一个文件里：

```
mpc_port/formation_cfg.m
```

常用参数：

| 参数名 | 当前值 | 含义 |
|---|---|---|
| `cfg.lead_v` | 1.5 m/s | 飞行速度（直线/圆周）|
| `cfg.lead_R` | 10 m | 圆周半径 |
| `cfg.lead_d` | 20 m | 直线飞行距离 |
| `cfg.target_alt` | -5.0 m | 飞行高度（NED坐标，负数=向上）|
| `cfg.d_safe` | 1.5 m | 无人机之间的安全距离 |
| `cfg.N` | 30 | MPC 预测步数（×0.05s = 1.5s 预测窗口）|

> **修改参数后直接重新运行 `run_mpc_*.m` 即可，不需要重启 MATLAB。**

## 常见问题

**Q：运行报错说找不到 `init.m`？**  
A：确认 MATLAB 当前目录在 `mpc_port/`，`init.m` 会自动从上级目录加载。

**Q：仿真结果和表格里的数字不一样？**  
A：正常。results.csv 是历史记录，重新跑会按最新参数计算。

**Q：`slprj/` 文件夹消失了？**  
A：没关系，它是编译缓存，下次运行会自动重新生成。

**Q：想只看某几个场景的动画？**  
A：直接运行 `animate_mpc('pair2', 'circle')` 即可，不用跑 `animate_all`。
